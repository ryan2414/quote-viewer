| 레벨 | 기준 |

## A. 단기 기능 — 서버 없이 구현 가능

### A-01. 명언 검색 (클라이언트 사이드)

| 항목 | 내용 |
|------|------|
| **우선순위** | High |
| **난이도** | 낮음 (반나절~1일) |
| **상태** | ✅ 구현 완료 (Phase 1, 2026-05-31) |

**Why (왜 필요한가)**
52개 명언이 100개, 200개로 늘어날수록 원하는 명언을 찾기 어렵다. 검색은 콘텐츠 탐색의 기본 UX다.

**What (무엇을 제공하는가)**
- 명언 본문, 저자명을 실시간으로 검색
- 검색어 강조 표시(하이라이트)
- 결과 없음 시 안내 메시지

**How (구현 방식)**
- `useState`로 검색어 관리, `useMemo`로 필터링
- 디바운스 없이 즉각 필터링 (데이터 규모 작음)
- `components/QuotesClient.tsx` 내 검색 입력 처리

**What**

- 클릭마다 무작위 명언 표시

**How (구현 방식)**
- `TodayQuoteSection.tsx`에서 `Math.random()` 기반 셔플
- 현재 표시 명언 제외 후 무작위 선택

## B. 중기 기능 — 백엔드 추가 필요

### B-01. 사용자 계정 및 즐겨찾기 동기화

| 항목 | 내용 |
|------|------|
| **우선순위** | Medium |
| **난이도** | 높음 (1~2주) |
| **상태** | ✅ 구현 완료 (Phase 4, 2026-06-11) |

**Why**
localStorage는 기기/브라우저 종속적이다. 즐겨찾기가 쌓일수록 "기기 바꿨더니 다 사라짐" 이탈 요인이 된다. 클라우드 동기화는 프리미엄 전환의 핵심 가치다.

**What**
- 소셜 로그인 (구글/카카오)
- 즐겨찾기 서버 저장 및 기기 간 동기화
- 사용자 프로필 (방문 기록, 통계)

**How**
- **Auth**: Supabase Auth + 구글 OAuth
- **DB**: Supabase (기존 연동)
- **API**: Next.js Route Handlers (`app/api/`)
- 마이그레이션: localStorage 즐겨찾기 → 로그인 시 서버로 자동 이전

---

### B-02. 명언 좋아요 카운트 (공개 인기도)

| 항목 | 내용 |
|------|------|
| **우선순위** | Medium |
| **난이도** | 중간 (3~5일, DB 필요) |
| **상태** | ✅ 구현 완료 (Phase 3, 2026-06-08) |

**Why**
"이 명언을 2,847명이 좋아했습니다" — 사회적 증거(Social Proof)는 신규 사용자 참여를 유도한다.

**What**
- 명언별 좋아요 수 표시 + 토글
- 비로그인 사용자도 가능

**How (구현 방식)**
- `app/api/likes/[id]/route.ts` — GET(카운트+liked), POST(토글)
- httpOnly 쿠키 `qv-did`로 익명 디바이스 ID 관리 (중복 방지)
- 낙관적 업데이트: `hooks/useLike.ts` — 실패 시 원상복구
- Supabase `quote_likes(quote_id, device_id)` UNIQUE 제약

> **보안 노트**: 쿠키 기반 디바이스 ID는 완벽한 중복 방지가 아님. 쿠키 삭제 시 재좋아요 가능. 강한 중복 방지가 필요하면 로그인 연동(B-01) 후 고도화 필요.

### C-02. 이메일/푸시 알림 (매일 오전 명언)

| 항목 | 내용 |
|------|------|
| **우선순위** | Medium |
| **난이도** | 중간 (3~5일) |
| **상태** | ✅ 구현 완료 (Phase 3, 2026-06-08) |

**Why**
"매일 아침 8시에 오늘의 명언을 이메일로 받는다" — 핵심 습관 형성 루프. DAU를 구조적으로 높이는 가장 강력한 메커니즘.

**What**
- 이메일 구독 (Footer에 폼)
- 매일 Vercel Cron으로 자동 발송
- 브라우저 푸시 알림 (VAPID 기반)
- 개인화된 구독 취소 링크

**How (구현 방식)**
- **이메일**: Resend API + `resend.batch.send()` 개별 발송
- **스케줄링**: Vercel Cron → `POST /api/newsletter/send` (CRON_SECRET 인증)
- **구독 취소 보안**: `unsubscribe_token` (UUID) 발급 → 링크 클릭 시 email+token 검증
- **푸시**: Web Push API + VAPID 키, `auth_token` 발급으로 구독 해제 소유권 확인
- Supabase: `newsletter_subscribers`, `push_subscriptions` 테이블

> **보안 노트**: 뉴스레터 발송은 `timingSafeEqual`로 타이밍 공격 방지. `CRON_SECRET` 환경 변수 필수 설정.
