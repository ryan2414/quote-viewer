# Quote Viewer 콘텐츠 전략 기획서

**작성일**: 2026-05-31  
**기준 상태**: 52개 명언 / 5개 카테고리 (wisdom/motivation/life/courage/love)  
**목적**: 콘텐츠 자산을 체계적으로 확장하여 SEO, 재방문, 공유를 동시에 강화한다.

## Executive Summary

- 현재 52개 명언은 "날짜 기반 오늘의 명언" 기능에 최소 필요 수량이나, 검색 유입과 카테고리 탐색 경험을 위해 **최소 200개, 목표 500개**가 필요하다.
- 콘텐츠 확장의 핵심은 **품질 우선 큐레이션** — 유명인의 검증된 명언 위주로 수집하고 출처를 반드시 명시한다.
- 시즌/이벤트 콘텐츠는 검색 급증 시점에 SEO 트래픽을 집중적으로 획득하는 핵심 전략이다.
- 사용자 제출 콘텐츠는 규모 확장 이후 검토하며, 품질 게이팅을 반드시 거친다.

#### 2순위: 검증된 출처

- **영어 명언**: Brainyquote, Goodreads, Wikiquote (출처 교차 검증 필수)
- **한국어 명언**: 책 직접 참조, 국립국어원 자료
- **주의**: "인터넷에 돌아다니는 명언"은 오귀인(잘못된 저자 표기) 위험이 높으므로 반드시 출처 확인

### 1.4 품질 기준 체크리스트

명언 추가 시 아래 기준을 모두 충족해야 한다.

- [ ] **진위 확인**: 해당 인물이 실제로 한 말인지 2개 이상 출처에서 확인
- [ ] **저자 표기**: 실명 또는 "알 수 없음" 명시 (익명 처리 금지)
- [ ] **한국어 품질**: 직역체가 아닌 자연스러운 한국어
- [ ] **중복 제거**: 기존 명언과 유사 의미 중복 방지
- [ ] **길이**: 한 화면에 읽기 적합한 길이 (3줄 이내 권장)
- [ ] **카테고리 적합성**: 지정한 카테고리와 내용 일치

### 1.5 데이터 구조 개선 제안

현재 구조에서 점진적으로 추가할 필드:

```typescript
// 현재
interface Quote {
  id: number
  text: string
  author: string
  category?: string
}

// 개선 방향 (하위 호환)
interface Quote {
  id: number
  text: string
  text_en?: string     // 영어 원문 (한국어 번역 명언인 경우)
  author: string
  source?: string      // 출처 (책명, 연설 등)
  year?: number        // 발언 연도
  category?: CategoryId
  tags?: string[]      // 검색/SEO를 위한 태그 (예: ['직장', '리더십'])
  isFeatured?: boolean // 특별 큐레이션 여부
}
```

**방법 1: 태그 기반 필터링 (권장)**
```typescript
interface Quote {
  // ...기존 필드
  tags?: string[]  // ['신년', '목표', '결심']
}

// 특정 날짜 범위에 해당 태그 명언 자동 표시
function getSeasonalQuotes(date: Date): Quote[] {
  const month = date.getMonth() + 1
  if (month === 1) return quotes.filter(q => q.tags?.includes('신년'))
  // ...
}
```

| 조건 | 기준 |
|------|------|
| 최소 명언 수 | 200개 이상 (큐레이션 기반 확립 후) |

### 4.3 제출 프로세스 설계

```
사용자 제출
  ↓
자동 검증 (욕설 필터, 중복 검사, 길이 제한)
  ↓
관리자 검수 대기열 (Supabase 테이블)
  ↓
승인/거절 (관리자 판단)
  ↓
승인 시: quotes 테이블에 추가, 제출자에게 알림
거절 시: 사유와 함께 이메일 안내
```

### 4.4 품질 관리 방안

| 방안 | 설명 | 구현 |
|------|------|------|

## 5. SEO 콘텐츠 전략

### 5.1 SEO 구조 설계

현재는 `/` (홈)과 `/quotes` (목록) 2개 페이지만 존재한다. SEO를 위해 콘텐츠 페이지 확장이 핵심이다.

#### 목표 URL 구조

```
/                           ← 홈 (오늘의 명언)
/quotes                     ← 전체 목록
/quotes/[id]                ← 명언 개별 페이지 ★ SEO 핵심
/category/[category]        ← 카테고리 페이지 ★
/author/[author]            ← 인물별 명언 페이지 ★
/events/[event]             ← 시즌 이벤트 페이지
```

### 5.2 명언 개별 페이지 (`/quotes/[id]`) SEO

```
페이지 제목: "당신의 시간은 제한되어 있다 — 스티브 잡스 명언 | Quote Viewer"
메타 디스크립션: "스티브 잡스의 명언: 당신의 시간은 제한되어 있다. 다른 사람의 삶을 사는 데 낭비하지 말라. — 카테고리: 인생"
OG 이미지: 해당 명언 카드 이미지 (동적 생성)
구조화 데이터: JSON-LD Quote 스키마
```

**구현 방법**
```typescript
// app/quotes/[id]/page.tsx
export async function generateMetadata({ params }) {
  const quote = quotes.find(q => q.id === Number(params.id))
  return {
    title: `${quote.text.slice(0, 30)}... — ${quote.author} 명언`,
    description: `${quote.author}의 명언: ${quote.text}`,
    openGraph: { images: [`/api/og?id=${params.id}`] }
  }
}

export function generateStaticParams() {
  return quotes.map(q => ({ id: String(q.id) }))
}
```

### 5.3 인물별 명언 페이지 (`/author/[author]`) SEO

**목표 키워드**: "스티브 잡스 명언", "간디 명언", "공자 명언"

- 이 키워드들은 월 수천~수만 건 검색량 보유
- 인물 소개 (2~3줄) + 해당 인물의 전체 명언 목록
- 관련 카테고리 링크

**구현**: `generateStaticParams`로 인물별 정적 페이지 생성

### 5.4 카테고리 페이지 (`/category/[category]`) SEO

**목표 키워드**: "동기부여 명언 모음", "인생 명언 모음", "용기 명언"

- 카테고리 설명 텍스트 (100~200자)
- 해당 카테고리 전체 명언 그리드
- 관련 인물 링크

### 5.5 내부 링크 구조

```
홈 → 오늘의 명언 개별 페이지
홈 → 목록 페이지
목록 페이지 → 각 명언 개별 페이지
명언 개별 페이지 → 같은 저자 명언
명언 개별 페이지 → 같은 카테고리 명언
카테고리 페이지 → 명언 개별 페이지들
인물 페이지 → 명언 개별 페이지들
```

**목적**: 크롤러가 모든 명언 페이지를 자연스럽게 발견하고 인덱싱

### 5.6 콘텐츠 SEO 강화 — 명언 부가 콘텐츠

각 명언 개별 페이지에 간단한 부가 설명 추가 시 체류 시간과 SEO 점수 향상:

```
오늘의 명언 | 스티브 잡스 (Steve Jobs)

"당신의 시간은 제한되어 있다..."

[저자 소개: 2-3줄 텍스트]
애플의 공동 창업자 스티브 잡스(1955-2011)는 혁신적인 기술과 디자인으로 
IT 산업을 변화시켰습니다. 이 명언은 2005년 스탠퍼드 대학교 졸업 연설에서...

[관련 명언 추천: 같은 저자 또는 같은 카테고리 3개]
```
