# Quote Viewer - Product Requirements Document

## 핵심 정보

**프로젝트명**: Quote Viewer  
**목적**: 매일 새로운 명언을 제공하여 사용자에게 영감과 동기부여를 주는 웹 서비스  
**대상 사용자**: 매일 짧은 명언을 통해 영감을 받고 싶은 일반 사용자  
**개발 기간**: MVP 기준 4단계  

---

## 사용자 여정

```
1. 홈 페이지 (명언 뷰어)
   ↓ 페이지 최초 진입
   오늘 날짜 기반 명언 자동 표시
   ↓ 사용자 선택
   [공유하기 클릭] → 클립보드 복사 완료 토스트 표시
   [즐겨찾기 클릭] → 즐겨찾기 페이지에 저장 (로컬스토리지)
   [목록 보기 클릭] → 명언 목록 페이지 이동
   ↓

2. 명언 목록 페이지
   ↓ 홈 페이지 목록 버튼 클릭 또는 헤더 메뉴 클릭
   전체 명언 카드 목록 표시
   ↓ 사용자 선택
   [명언 카드 클릭] → 홈 페이지로 이동 (선택 명언 표시)
   [즐겨찾기 탭 클릭] → 즐겨찾기 명언만 필터링 표시
   ↓

3. 즐겨찾기 탭 (명언 목록 페이지 내)
   ↓ 명언 목록 페이지에서 탭 전환
   즐겨찾기 저장된 명언만 표시
   [즐겨찾기 해제] → 목록에서 제거
   ↓

4. 완료 → 홈으로 돌아가기
```

## 기능 명세

### MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| **F001** | 오늘의 명언 표시 | 날짜 기반으로 명언 배열에서 하나를 선택해 표시 | 서비스의 핵심 가치 | 홈 페이지 |
| **F002** | 명언 목록 조회 | 전체 명언을 카드 형태로 목록 표시 | 콘텐츠 탐색 기능 | 명언 목록 페이지 |
| **F003** | 명언 즐겨찾기 | 명언을 로컬스토리지에 저장/해제 | 재방문 동기 부여 | 홈 페이지, 명언 목록 페이지 |
| **F004** | 명언 공유 | 클립보드에 명언 텍스트 복사 | 바이럴 확산 수단 | 홈 페이지 |

### MVP 필수 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| **F010** | 즐겨찾기 필터 | 즐겨찾기한 명언만 탭으로 필터링 | F003 기능 활용성 극대화 | 명언 목록 페이지 |
| **F011** | 명언 데이터 관리 | 정적 JSON 또는 TypeScript 배열로 명언 데이터 관리 | 핵심 콘텐츠 데이터 제공 | 홈 페이지, 명언 목록 페이지 |

### MVP 이후 기능 (범위 외)

- 사용자 인증 및 회원가입
- 서버 기반 즐겨찾기 동기화
- 명언 직접 등록/편집 기능
- 카테고리별 분류 및 고급 필터
- 실시간 알림 (오늘의 명언 푸시)
- 소셜 공유 (카카오, 트위터 연동)

## 페이지별 상세 기능

### 홈 페이지

> **구현 기능:** `F001`, `F003`, `F004` | **메뉴 위치:** 헤더 홈 메뉴

| 항목 | 내용 |
|------|------|
| **역할** | 오늘 날짜 기반의 명언을 대형 타이포그래피로 표시하는 메인 랜딩 페이지 |
| **진입 경로** | 최초 접속, 헤더 홈 메뉴 클릭, 명언 목록 페이지에서 명언 카드 클릭 |
| **사용자 행동** | 오늘의 명언을 읽고, 즐겨찾기에 저장하거나 클립보드로 복사해 공유 |
| **주요 기능** | - 오늘 날짜를 명언 배열 인덱스로 변환하여 명언 자동 선택 표시<br>- 명언 텍스트, 저자, 출처 표시<br>- 즐겨찾기 토글 버튼 (로컬스토리지 저장/해제, 하트 아이콘)<br>- **명언 공유** 버튼 클릭 시 클립보드 복사 후 토스트 메시지 표시<br>- 명언 목록 페이지로 이동하는 링크 버튼 |
| **다음 이동** | 즐겨찾기 클릭 → 현재 페이지 유지 (상태 토글), 공유 클릭 → 현재 페이지 유지 (토스트 표시), 목록 보기 클릭 → 명언 목록 페이지 |

---

### 명언 목록 페이지

> **구현 기능:** `F002`, `F003`, `F010`, `F011` | **메뉴 위치:** 헤더 명언 목록 메뉴

| 항목 | 내용 |
|------|------|
| **역할** | 전체 명언을 카드 형태로 탐색하고 즐겨찾기를 관리하는 페이지 |
| **진입 경로** | 헤더 명언 목록 메뉴 클릭, 홈 페이지 목록 보기 버튼 클릭 |
| **사용자 행동** | 전체 명언을 스크롤하며 탐색하고, 즐겨찾기 탭으로 저장한 명언만 확인하며 관리 |
| **주요 기능** | - 전체 탭 / 즐겨찾기 탭 전환 UI<br>- 전체 탭: 명언 배열 전체를 카드 그리드로 렌더링<br>- 즐겨찾기 탭: 로컬스토리지 즐겨찾기 ID 기반 필터링 후 카드 표시<br>- 각 카드에 즐겨찾기 토글 버튼 표시 (F003)<br>- 카드 클릭 시 홈 페이지로 이동하며 해당 명언 표시<br>- 즐겨찾기 탭에서 즐겨찾기 해제 시 목록에서 즉시 제거 |
| **다음 이동** | 카드 클릭 → 홈 페이지 (선택 명언 표시), 즐겨찾기 해제 → 현재 페이지 유지 (목록 갱신) |

## 기술 스택

### 프론트엔드 프레임워크

- **Next.js 16.2.6** (App Router) - 기존 프로젝트 스택 유지
- **React 19.2.4** - UI 라이브러리
- **TypeScript** - 타입 안전성 보장

### 스타일링 & UI

- **Tailwind CSS 4** - 유틸리티 CSS 프레임워크 (CSS 변수 기반 새 엔진)
- **Lucide React** - 하트, 공유, 복사 등 아이콘 라이브러리

### 상태 관리 & 데이터

- **React useState / useEffect** - 클라이언트 상태 관리
- **localStorage API** - 즐겨찾기 영속성 저장 (서버 불필요)
- **정적 TypeScript 배열** - 명언 데이터 (`data/quotes.ts`)

### 배포

- **Vercel** - Next.js 최적화 배포 플랫폼

## 개발 단계별 요구사항

### 1단계 - 데이터 및 타입 설정

- `types/quote.ts` 에 Quote 인터페이스 정의
- `data/quotes.ts` 에 최소 30개 명언 데이터 작성 (한국어/영어 혼합)
- 날짜 기반 명언 선택 유틸리티 함수 작성 (`(연도+월+일) % 명언총수` 인덱스)

### 2단계 - 핵심 컴포넌트 구현

- `useFavorites.ts` 커스텀 훅 구현 (로컬스토리지 읽기/쓰기, 토글 함수)
- `QuoteDisplay.tsx` 구현 (대형 타이포그래피, 저자, 즐겨찾기/공유 버튼)
- `QuoteCard.tsx` 구현 (카드 레이아웃, 즐겨찾기 버튼 포함)
- `Toast.tsx` 구현 (복사 완료 메시지, 3초 후 자동 사라짐)

### 3단계 - 페이지 구현

- `app/page.tsx` 홈 페이지 구현 (오늘의 명언, F001, F003, F004)
- `app/quotes/page.tsx` 명언 목록 페이지 구현 (탭 전환, F002, F010)
- `app/layout.tsx` 헤더 네비게이션 추가

### 4단계 - 스타일링 및 마무리

- Tailwind CSS 4로 반응형 레이아웃 완성 (모바일 우선)
- 다크 모드 지원 (`dark:` 프리픽스 활용)
- `npm run build` 빌드 오류 없음 확인
- `npm run lint` 코드 품질 확인
