# Quote Viewer 개발 로드맵

매일 새로운 명언으로 사용자에게 영감과 동기부여를 주는 웹 서비스

## 개요

Quote Viewer는 매일 짧은 명언을 통해 영감을 받고 싶은 일반 사용자를 위한 명언 뷰어 서비스로 다음 기능을 제공합니다:

- **오늘의 명언 표시 (F001)**: 오늘 날짜를 기반으로 명언 배열에서 하나를 선택해 대형 타이포그래피로 표시
- **명언 목록 조회 (F002)**: 전체 명언을 카드 그리드 형태로 탐색
- **명언 즐겨찾기 (F003 / F010)**: 명언을 로컬스토리지에 저장/해제하고 즐겨찾기 탭으로 필터링
- **명언 공유 (F004)**: 명언 텍스트를 클립보드에 복사하고 토스트로 안내

> **참고**: 본 프로젝트는 서버 없이 정적 TypeScript 데이터와 localStorage만 사용하는 클라이언트 중심 MVP입니다. 따라서 별도의 데이터베이스/인증 Phase 대신 **로컬 상태 로직과 영속성 검증**에 Playwright MCP 테스트를 집중합니다.

## 개발 워크플로우

1. **작업 계획**
   - 기존 코드베이스를 학습하고 현재 상태를 파악
   - 새로운 작업을 포함하도록 `ROADMAP.md` 업데이트
   - 우선순위 작업은 마지막 완료된 작업 다음에 삽입

2. **작업 생성**
   - 기존 코드베이스를 학습하고 현재 상태를 파악
   - 고수준 명세서, 관련 파일, 수락 기준, 구현 단계 포함
   - 비즈니스 로직/영속성 작업 시 "## 테스트 체크리스트" 섹션 필수 포함 (Playwright MCP 테스트 시나리오 작성)
   - 새 작업의 경우 문서에는 빈 박스와 변경 사항 요약이 없어야 함

3. **작업 구현**
   - 작업 파일의 명세서를 따름
   - 기능과 기능성 구현
   - 비즈니스 로직(날짜 기반 선택, 즐겨찾기 토글, 클립보드 복사) 구현 시 Playwright MCP로 테스트 수행 필수
   - 각 단계 후 작업 파일 내 단계 진행 상황 업데이트
   - 구현 완료 후 Playwright MCP를 사용한 E2E 테스트 실행
   - 테스트 통과 확인 후 다음 단계로 진행
   - 각 단계 완료 후 중단하고 추가 지시를 기다림

4. **로드맵 업데이트**
   - 로드맵에서 완료된 작업을 ✅로 표시

## 기술 스택

- **Next.js 16.2.6** (App Router) — React 프레임워크
- **React 19.2.4** — UI 라이브러리
- **TypeScript** — 타입 안전성
- **Tailwind CSS 4** — CSS 변수 기반 새 엔진
- **Lucide React** — 하트/공유/복사 아이콘
- **localStorage API** — 즐겨찾기 영속성 (서버 불필요)
- **Vercel** — 배포 플랫폼

## 개발 단계

### Phase 1: 데이터 및 골격 구축

> 목표: 전체 라우트 구조, 타입, 데이터를 먼저 확정하여 이후 Phase에서 UI/로직 팀이 병렬로 작업 가능하도록 한다.

- **Task 001: 프로젝트 구조 및 라우팅 골격 설정** - 우선순위
  - `app/quotes/` 라우트 디렉토리 및 빈 `page.tsx` 껍데기 생성
  - `components/`, `data/`, `hooks/`, `types/` 디렉토리 구조 생성
  - 홈(`app/page.tsx`)을 빈 명언 뷰어 골격으로 교체 (기본 create-next-app 제거)
  - `app/layout.tsx` 헤더 자리(placeholder) 및 메타데이터(title/description) 갱신
  - `lucide-react` 패키지 설치
  - **수락 기준**: `npm run dev` 실행 시 `/`, `/quotes` 두 경로가 빈 페이지로 정상 렌더링됨

- **Task 002: 타입 정의 및 데이터 모델 설계**
  - `types/quote.ts` 에 `Quote` 인터페이스 정의 (`id`, `text`, `author`, `source?`, `category?`)
  - 즐겨찾기 저장 구조(`FavoriteStore`: `favoriteIds: number[]`) 타입 정의
  - 컴포넌트 Props 타입(예: `QuoteCardProps`, `QuoteDisplayProps`) 시그니처 선언
  - **수락 기준**: `npm run build` 시 타입 에러 없음, 모든 데이터/컴포넌트가 참조할 단일 타입 소스 확보

- **Task 003: 명언 데이터 및 날짜 유틸리티 작성**
  - `data/quotes.ts` 에 최소 30개 명언 데이터 작성 (한국어/영어 혼합, `id` 고유)
  - 날짜 기반 명언 선택 유틸 `getTodayQuote()` 작성 — `(연도+월+일) % 명언총수` 인덱스 규칙
  - `id` 로 명언을 조회하는 `getQuoteById()` 유틸 작성 (목록→홈 이동 시 사용)
  - **수락 기준**: 임의 날짜 입력 시 결정론적으로 동일 인덱스 반환, 30개 이상 데이터 확보

  #### 테스트 체크리스트
  - 동일 날짜 입력 시 항상 같은 명언이 선택되는지 검증 (결정론성)
  - 날짜가 바뀌면 다른 인덱스가 선택되는지 검증
  - 명언 총수보다 큰 날짜 합산값에 대해 인덱스가 배열 범위를 벗어나지 않는지 검증 (모듈러 경계)
  - 존재하지 않는 `id` 조회 시 안전하게 처리(undefined 반환)되는지 검증

### Phase 2: 핵심 컴포넌트 구현 (UI + 클라이언트 로직)

> 목표: 더미/정적 데이터를 사용해 재사용 컴포넌트와 클라이언트 상태 훅을 완성한다.

- **Task 004: useFavorites 커스텀 훅 구현** - 우선순위
  - `hooks/useFavorites.ts` — localStorage 읽기/쓰기 래핑
  - `favoriteIds` 상태, `toggleFavorite(id)`, `isFavorite(id)` 함수 노출
  - SSR 안전 처리 (마운트 후 localStorage 접근, hydration mismatch 방지)
  - 다른 컴포넌트/탭 간 상태 동기화 고려 (`storage` 이벤트 또는 단일 소스)
  - **수락 기준**: 새로고침 후에도 즐겨찾기 상태 유지

  #### 테스트 체크리스트
  - 즐겨찾기 추가 후 새로고침 시 유지되는지 E2E 검증 (영속성)
  - 즐겨찾기 추가→해제 토글이 localStorage에 정확히 반영되는지 검증
  - localStorage가 비어있는 최초 진입 시 빈 배열로 안전 초기화되는지 검증
  - SSR 환경에서 hydration 에러/콘솔 경고가 없는지 검증

- **Task 005: QuoteDisplay 컴포넌트 구현 (홈용)**
  - `components/QuoteDisplay.tsx` — 대형 타이포그래피로 명언 본문/저자/출처 표시
  - 즐겨찾기 토글 버튼 (하트 아이콘, `useFavorites` 연동)
  - 공유 버튼 (클립보드 복사 트리거, 콜백 prop으로 토스트 호출)
  - "목록 보기" 링크 버튼 (`/quotes` 이동)
  - **수락 기준**: props로 받은 명언을 정확히 렌더링, 버튼 인터랙션 동작

- **Task 006: QuoteCard 컴포넌트 구현 (목록용)**
  - `components/QuoteCard.tsx` — 카드 레이아웃(본문/저자) + 즐겨찾기 토글 버튼
  - 카드 클릭 시 홈으로 이동하며 해당 명언 표시 (id 쿼리/라우팅 처리)
  - 즐겨찾기 버튼 클릭이 카드 클릭 이벤트와 분리되도록 처리 (이벤트 전파 차단)
  - **수락 기준**: 그리드 내 카드가 올바른 명언을 표시하고 두 인터랙션이 독립 동작

- **Task 007: Toast 및 FavoriteButton 컴포넌트 구현**
  - `components/Toast.tsx` — 복사 완료 메시지, 3초 후 자동 사라짐 (타이머 정리 포함)
  - `components/FavoriteButton.tsx` — 즐겨찾기 토글 공통 버튼 (하트 채움/빈 상태)
  - 접근성: 버튼 `aria-label`, 토스트 `role="status"` 적용
  - **수락 기준**: 토스트 표시 후 3초 뒤 자동 제거, 즐겨찾기 버튼 상태 시각적 반영

  #### 테스트 체크리스트
  - 공유 버튼 클릭 시 클립보드에 명언 텍스트가 복사되는지 검증 (clipboard API)
  - 복사 직후 토스트가 노출되고 3초 후 사라지는지 검증
  - 토스트가 빠르게 연속 트리거될 때 타이머가 정상 정리되는지 검증

### Phase 3: 페이지 통합 및 사용자 플로우 완성

> 목표: 완성된 컴포넌트와 데이터를 페이지에 결합하여 PRD 사용자 여정을 실제로 동작시킨다.

- **Task 008: 홈 페이지 구현 (F001, F003, F004)** - 우선순위
  - `app/page.tsx` — `getTodayQuote()`로 오늘의 명언 선택 후 `QuoteDisplay` 렌더링
  - 즐겨찾기 토글 연동 (F003), 공유 클릭 시 클립보드 복사 + Toast 표시 (F004)
  - 목록에서 카드 클릭으로 넘어온 경우 해당 명언 우선 표시 (id 파라미터 처리)
  - **수락 기준**: 진입 시 날짜 기반 명언 표시, 즐겨찾기/공유/목록이동 모두 동작

  #### 테스트 체크리스트
  - 홈 진입 시 오늘 날짜에 해당하는 명언이 표시되는지 E2E 검증
  - 즐겨찾기 클릭 → 상태 토글 → 새로고침 후 유지 검증
  - 공유 클릭 → 클립보드 복사 → 토스트 노출 전체 플로우 검증
  - "목록 보기" 클릭 시 `/quotes`로 정상 이동하는지 검증

- **Task 009: 명언 목록 페이지 구현 (F002, F003, F010)**
  - `app/quotes/page.tsx` — 전체 탭 / 즐겨찾기 탭 전환 UI
  - 전체 탭: 명언 배열 전체를 카드 그리드로 렌더링 (F002)
  - 즐겨찾기 탭: `useFavorites`의 ID 기반 필터링 후 카드 표시 (F010)
  - 즐겨찾기 탭에서 해제 시 목록에서 즉시 제거, 카드 클릭 시 홈 이동
  - 즐겨찾기가 비었을 때 빈 상태(empty state) UI 표시
  - **수락 기준**: 탭 전환 및 필터링 정상, 카드 클릭으로 홈 이동 및 명언 표시

  #### 테스트 체크리스트
  - 전체 탭에서 30개 이상 명언 카드가 렌더링되는지 검증
  - 즐겨찾기 탭이 저장된 명언만 필터링해 표시하는지 검증
  - 즐겨찾기 탭에서 해제 시 목록에서 즉시 사라지는지 검증
  - 카드 클릭 시 홈으로 이동하며 해당 명언이 표시되는지 검증
  - 즐겨찾기가 없을 때 빈 상태 UI가 노출되는지 검증

- **Task 010: 헤더 네비게이션 및 레이아웃 통합**
  - `components/Header.tsx` — 홈 / 명언 목록 네비게이션 링크
  - `app/layout.tsx`에 헤더 마운트, 현재 경로 활성 표시(active state)
  - 모바일 네비게이션 동작 확인
  - **수락 기준**: 모든 페이지 상단에 헤더 노출, 메뉴로 페이지 간 이동 가능

- **Task 010-1: 핵심 사용자 플로우 통합 테스트**
  - Playwright MCP로 PRD 사용자 여정 전체 시나리오 E2E 테스트
  - 홈 → 즐겨찾기 → 목록 → 즐겨찾기 탭 → 카드 클릭 → 홈 복귀 플로우 검증
  - 영속성/엣지 케이스(빈 즐겨찾기, 새로고침, 잘못된 id) 검증
  - **수락 기준**: 전체 사용자 여정이 끊김 없이 동작, 콘솔 에러 없음

  #### 테스트 체크리스트
  - 홈→즐겨찾기 저장→목록 즐겨찾기 탭에 노출되는 end-to-end 시나리오 검증
  - 즐겨찾기 탭에서 해제 후 홈으로 돌아가 상태가 일관되는지 검증
  - 헤더 네비게이션을 통한 모든 경로 이동 검증
  - 새로고침/직접 URL 진입 시 상태와 라우팅이 깨지지 않는지 검증

### Phase 4: 스타일링, 다크 모드 및 배포

> 목표: 반응형/다크 모드를 완성하고 빌드·린트·배포 품질을 확정한다.

- **Task 011: 반응형 레이아웃 및 디자인 마무리**
  - Tailwind CSS 4로 모바일 우선 반응형 레이아웃 완성 (`sm:`, `md:`, `lg:`)
  - 명언 목록 카드 그리드 반응형 컬럼 구성
  - 타이포그래피/여백/색상 디자인 시스템 일관성 적용
  - **수락 기준**: 모바일·태블릿·데스크톱에서 레이아웃 깨짐 없음

- **Task 012: 다크 모드 지원**
  - `dark:` 프리픽스 기반 전역 다크 모드 스타일 적용

  - 토스트/카드/버튼 등 모든 컴포넌트 다크 모드 대비 확인
  - **수락 기준**: 시스템 다크 모드 전환 시 모든 화면 정상 표시

- **Task 013: 빌드·린트 검증 및 배포**
  - `npm run lint` 코드 품질 통과
  - `npm run build` 빌드 오류 없음 확인
  - Vercel 배포 및 프로덕션 동작 점검
  - **수락 기준**: 린트/빌드 무오류, Vercel 배포 성공
